# 如何节省网站的流量 

左岸终于两天不能访问了，原因大家都猜到了——左岸这个月爆了流量[bandwidth]。Jiang先生已 经给我加了三次的流量，流量还一直在爆。不能否认的是这个月的访问有增长的趋势，但涨到这么多，是超出我的承受范围的，因为国外的主机基本都有流量限制， 超出的话要么像盘古把你网站给关了，要么像dreamhost 追加高昂的流量费，要么像我用的这个主机一样直接挂掉，所以流量爆了都是很惨的事，这里参考煎蛋的一些方法，思考一下如何节省网站的流量，节省流量省的就是money（如果再这样下去，左岸就不能做到平衡了）！
1- 使用外部图片存储
这个我做到了，我原来都把图片放在http://picasaweb.google.com,现在不行了，picasa在中国暂时也挂了，我就把图片放到了photobucket
当然你还可以有多重选择：比如Imageshack / Flickr / Amazon S3
2- 使用 HTTP 压缩
据说能节省66%的流量，可以在这里测试具体网站用了HTTP 压缩的话能节约多少。左岸首页未压缩是51534 bytes，压缩后是17091 bytes，压缩了67%，访问速度将快3.0倍。
我想问题可能出在这个上面了，因为我用的是WP Super Cache缓存插件，原来启用了里面的选项Super Cache Compression，现在却不支持了，所以流量爆涨。我个看能不能从Jiang先生那儿获得技术支持。
3- 将 RSS feed 烧录输出
提供 RSS 烧录的，知名的有国外的 FeedBurner和国内的 Feedsky 。Feedburner 功能十分强大，前不久推出了feed广告和network功能；而 Feedsky 的好处是可以自定义域名和页面模板。
4- 优化 JS 和 CSS 代码
相信这几招大家都知道，但是的确很实用。春节前知名博客王小峰同学不是也流量爆了被迫把 feed 输出给关了吗，如果他早点烧录到 Feedburner 就不会那么被动了。Sina 的博客一会不能显示图片一会又折腾成摘要输出，门户都被逼的这么不厚道了，可见节省流量是很重要的。
